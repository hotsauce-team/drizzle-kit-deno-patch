Compare the work on the current branch to main.

Peer review the code, and write the review to REVIEW.md

## What to Review

- Code correctness and logic
- Test coverage
- Documentation updates
- Breaking changes
- Security considerations

## Review Format

Use these sections:
1. **Summary** - Brief overview of the changes
2. **Changes Reviewed** - Detailed review per file/area
3. **Checklist** - Actionable fixes (if any)
4. **Verdict** - LGTM, needs changes, etc.

## Checklist Items

Where there are actionable fixes, make a checklist. After writing the review, ask whether to perform the fixes automatically.
